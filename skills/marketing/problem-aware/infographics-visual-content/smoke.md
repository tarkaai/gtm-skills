# Infographics & Visual Content — Smoke Test

> **Stage:** Marketing → Problem Aware | **Motion:** Founder Social Content | **Channels:** Social, Content

## Overview
Create 1-2 data-driven infographics to test whether visual content resonates with your ICP and generates shares. This is a manual, local agent run. No automation, no always-on systems. Produce the infographic, post it, measure the signal.

**Time commitment:** 6 hours over 1 week
**Pass threshold:** ≥100 impressions and ≥10 shares in 3 days

---

**Play-specific cost:** Free

All rendering uses open-source tools (Plotly, Satori, Sharp). LLM costs for spec generation are under $0.05 total.

## Instructions

### 1. Define your visual content ICP
Run the `icp-definition` drill to define who you are creating infographics for. Document: target audience job titles, the data topics they care about, which social platforms they use most, and what format of visual content gets shared in their feeds (charts, stat cards, process diagrams, comparison tables).

### 2. Source data for your first infographic
Identify one data point or dataset that your ICP would find surprising, useful, or validating. Sources to check:
- Your own product analytics (anonymized and aggregated)
- Industry reports (Gartner, McKinsey, CB Insights, state-of-X surveys)
- Public datasets (Bureau of Labor Statistics, GitHub State of Octoverse, Stack Overflow Developer Survey)
- Original research (run a quick poll on LinkedIn or Twitter, collect 50+ responses)

The data must be from a credible, citeable source. Do not fabricate statistics.

### 3. Create 1-2 infographics
Run the the infographic creation pipeline workflow (see instructions below) drill to produce the infographic. For the Smoke Test:
- Generate the infographic spec via LLM (provide raw data + topic angle + ICP description)
- Render the data visualization using Plotly or Chart.js (free, open source)
- Compose the final infographic using Satori or Matplotlib (free, open source)
- Export for LinkedIn (1200x1200) and Twitter (1200x675) using Sharp or Pillow

**Human action required:** Review the rendered infographic before posting. Verify all statistics match the source data. Check that text is readable at mobile size. Confirm your brand colors and logo are correctly placed.

### 4. Post and track manually
**Human action required:** Post the infographic to LinkedIn and Twitter manually. Use the companion copy generated by the pipeline drill. Post on Tuesday-Thursday between 8-10am in your ICP's timezone.

Track manually in a spreadsheet or Attio:
- Impressions (check 24h and 72h after posting)
- Shares/reposts
- Comments
- Profile views
- DMs received
- Any inbound links or mentions

### 5. Evaluate against threshold
Run the `threshold-engine` drill to measure results against: ≥100 impressions and ≥10 shares in 3 days. If PASS, proceed to Baseline. If FAIL, diagnose: was the topic boring, the visual design unclear, the data not surprising enough, or was it posted at a bad time? Adjust and create a second infographic with the learnings.
