# Infographics & Visual Content — Durable Intelligence

> **Stage:** Marketing → Problem Aware | **Motion:** Founder Social Content | **Channels:** Social, Content

## Overview
Deploy always-on AI agents that autonomously monitor infographic performance, detect anomalies, generate improvement hypotheses, run experiments, and implement winners. The `autonomous-optimization` drill runs the core optimization loop. The `autonomous-optimization` drill provides the infographic-specific monitoring, attribution, and reporting that feeds the optimization loop with actionable data.

**Time commitment:** 180 hours over 12 months
**Pass threshold:** Sustained backlink growth (≥15/month) over 12 months via AI-driven topic selection and design optimization

## Instructions

### 1. Deploy the infographic performance monitor
Run the `autonomous-optimization` drill to build the always-on monitoring system:

- **PostHog dashboard** with 5 panels: publishing cadence, social engagement, backlink acquisition, traffic/pipeline, and efficiency metrics
- **Anomaly detection** checking daily for engagement drops and weekly for backlink drought, link quality decline, or traffic drops
- **Automated backlink sync** from Ahrefs running weekly via n8n
- **Weekly performance reports** generated by Claude and posted to Slack
- **Monthly topic effectiveness analysis** ranking topics by engagement, backlinks, and leads

This monitoring system produces the data that the autonomous optimization loop consumes.

### 2. Deploy the autonomous optimization loop
Run the `autonomous-optimization` drill configured for the infographic play. The optimization loop runs continuously:

**Phase 1 — Monitor (daily):** Check infographic KPIs against rolling averages. Classify as normal, plateau, drop, or spike. If anomaly detected, trigger diagnosis.

**Phase 2 — Diagnose (triggered by anomaly):** Pull the current infographic strategy configuration (topic mix, visual formats, posting schedule, outreach templates) from Attio. Pull 8-week metric history from PostHog. Generate 3 ranked hypotheses via Claude for what to change.

**Phase 3 — Experiment (triggered by hypothesis):** Design and run an A/B test via PostHog feature flags. Examples of experiments the loop might run:
- **Topic experiment:** Test a new content pillar against the current best performer
- **Format experiment:** Test carousel vs single image for the same data
- **Timing experiment:** Test posting on a different day or time
- **Outreach experiment:** Test a new email subject line or value proposition
- **Visual style experiment:** Test a new color scheme or chart type
- **CTA experiment:** Test different companion post CTAs

Run each experiment for minimum 7 days or 6+ infographics, whichever is longer.

**Phase 4 — Evaluate (triggered by experiment completion):** Pull results from PostHog. Run evaluation via Claude. Adopt the winner, iterate if promising but not significant, or revert if the variant lost.

**Phase 5 — Report (weekly):** Generate optimization briefs covering: anomalies detected, hypotheses tested, experiments run, results, net metric impact, and distance from local maximum.

### 3. Configure infographic-specific optimization targets

Set these as the optimization targets the autonomous loop works toward:

| Metric | Target | Optimization Lever |
|--------|--------|--------------------|
| Shares per infographic | ≥15 average | Topic selection, visual format, posting time |
| Backlinks per infographic | ≥2 average | Outreach targeting, email copy, data uniqueness |
| Engagement rate | ≥5% average | Visual design, hook quality, CTA style |
| Outreach reply rate | ≥10% | Email personalization, target selection, timing |
| Cost per backlink | ≤$25 | Outreach efficiency, organic link rate improvement |
| Blog traffic per post | ≥200 visits/month | SEO optimization, internal linking, embed code usage |

### 4. Enable autonomous topic discovery

Build an n8n workflow that feeds new topic opportunities to the creation pipeline:

1. Monitor RSS feeds, industry reports, and trending topics APIs weekly
2. Score each topic against ICP relevance and data availability
3. Check whether the topic overlaps with an existing high-performing pillar
4. If score exceeds threshold and data is available, add to the production queue
5. If a current topic pillar shows 3+ weeks of declining engagement, flag for retirement

The agent should maintain a rolling topic backlog of 10-15 scored, data-ready topics. The creation pipeline draws from this backlog.

### 5. Enable autonomous outreach optimization

Configure the optimization loop to experiment on outreach independently:

- Test new outreach target segments (different blog categories, different DR ranges)
- Test email copy variations (data-first vs story-first vs question-first)
- Test send timing (day of week, time of day)
- Auto-retire outreach templates with <3% reply rate after 50+ sends
- Auto-promote templates with >15% reply rate to primary position

### 6. Run monthly strategic reviews

The agent generates a monthly strategic review covering:
- **Topic effectiveness ranking:** topics by backlinks, shares, and leads (retire bottom 20%, double down on top 20%)
- **Format effectiveness:** which visual formats to produce more or less of
- **Outreach health:** reply rate trends, new link partner acquisition rate, domain rating quality trend
- **Competitive landscape:** are competitors producing similar infographics? If so, differentiate by data source or visual quality
- **Convergence assessment:** has the play reached its local maximum? If successive experiments produce <2% improvement for 3 consecutive experiments, report convergence

**Human action required:** Review the monthly strategic review. Approve recommended topic retirements and additions. Decide whether to expand to new platforms or data categories. Provide new data sources or partnerships the agent cannot discover autonomously.

### 7. Handle convergence

When the autonomous loop detects convergence (3 consecutive experiments with <2% improvement):
1. Reduce monitoring frequency from daily to weekly
2. Reduce experiment frequency from weekly to monthly
3. Report: "Infographic play has reached local maximum. Current performance: [metrics]. Further gains require strategic changes: new data partnerships, new platforms (YouTube data videos, podcast data segments), or new audience segments."
4. Maintain the production pipeline at the optimized cadence
5. Continue backlink monitoring to detect and respond to link loss
