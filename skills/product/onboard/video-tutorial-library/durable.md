# Video Tutorial Library — Durable

> **Stage:** Product > Onboard | **Motion:** LeadCaptureSurface | **Channels:** Product, Email, Content

## Outcomes

The video tutorial library runs as an always-on system with autonomous optimization. An AI agent monitors per-video and per-persona metrics daily, detects when any metric plateaus or declines, generates hypotheses for improvement, runs A/B experiments, evaluates results, and auto-implements winners. Human involvement is limited to recording new video content when the agent identifies gaps, and reviewing the weekly optimization brief. The system sustains or improves the 6pp activation lift achieved at Scalable level for 6+ months.

## Leading Indicators

- Daily health monitor runs without failures for 30+ consecutive days
- Anomalies are detected within 24 hours of metric shifts
- Hypotheses generated by the agent are actionable (>50% lead to launchable experiments)
- At least 2 experiments complete per month with valid sample sizes
- Winning experiments produce measurable improvement (>2% on target metric)
- Weekly optimization briefs are generated on schedule with accurate data
- Content freshness score stays above 80% (fewer than 20% of tutorials are stale)
- No metric drops below the Scalable threshold for more than 2 consecutive weeks without the agent detecting and responding

## Instructions

### 1. Deploy the video tutorial health monitor

Run the `video-tutorial-health-monitor` drill to set up:

1. The video library health dashboard in PostHog (10 panels covering play rate, completion, activation, per-video scores, surface effectiveness, persona engagement, and content freshness)
2. Per-video health scoring (0-100 scale: engagement + retention + impact)
3. Daily monitoring workflow in n8n running at 07:00 UTC
4. Content freshness tracking (flag tutorials where the product UI has changed since recording)
5. Coverage gap detection (identify onboarding paths and search queries with no matching tutorial)
6. Weekly health report generation every Monday at 09:00 UTC

Verify the health monitor runs correctly for 1 week before enabling autonomous optimization.

### 2. Deploy the autonomous optimization loop

Run the `autonomous-optimization` drill configured for the video tutorial library:

**Phase 1 — Monitor (daily via n8n cron):**
- The agent checks the video tutorial library's primary KPIs: play rate, completion rate, and post-video activation rate
- Compares last 2 weeks against 4-week rolling average
- Classifies: normal (within 10%), plateau (within 2% for 3+ weeks), drop (>20% decline), spike (>50% increase)
- If normal, log to Attio. If anomaly, trigger Phase 2

**Phase 2 — Diagnose (triggered by anomaly):**
- The agent gathers context: which videos dropped, which personas are affected, which surfaces show the change
- Pulls data from the video tutorial health monitor's per-video scores
- Generates 3 ranked hypotheses using Claude. Example hypotheses for this play:
  - "Play rate dropped because Intercom in-app message frequency was reduced by a product change. Test restoring the previous frequency."
  - "Completion rate on the 'Data Import' tutorial dropped because the UI changed and the tutorial no longer matches the current product. Re-record the tutorial."
  - "Post-video activation rate dropped for the Analyst persona because the CTA links to the old dashboard page. Update the deep link."
- If hypothesis risk = high (e.g., requires re-recording multiple videos), alert human. Otherwise, proceed.

**Phase 3 — Experiment (triggered by hypothesis acceptance):**
- The agent creates a PostHog experiment with feature flag
- For testable hypotheses (CTA text, thumbnail, placement, trigger timing, email subject line), the agent implements both variants
- For content hypotheses (video too long, hook not compelling), the agent flags for human re-recording and queues the experiment to start when the new version is ready
- Minimum experiment duration: 7 days or 100 users per variant, whichever is longer

**Human action required:** When the agent identifies a hypothesis that requires new video content (re-recording, new tutorial for a gap), a human must record the video. The agent prepares the outline, specifies the target duration and CTA, and sets up the experiment infrastructure. Once the human uploads the new Loom video, the agent activates the experiment.

**Phase 4 — Evaluate (triggered by experiment completion):**
- The agent pulls results from PostHog
- Evaluates statistical significance and practical impact
- Adopt: implement the winner permanently, update the routing matrix
- Iterate: generate a refined hypothesis and return to Phase 2
- Revert: restore the control, log the failure, return to Phase 1
- Extend: keep running if sample size is insufficient

**Phase 5 — Report (weekly via n8n cron):**
- Generate the weekly optimization brief:

```
# Video Tutorial Optimization Brief — Week of {date}

## This Week
- Anomalies detected: {N}
- Hypotheses generated: {N}
- Experiments started: {N}
- Experiments completed: {N}
- Decisions: {N adopted, N reverted, N extended}

## Metric Summary
- Play rate: {X}% (4-week avg: {Y}%, change: {delta})
- Completion rate: {X}% (4-week avg: {Y}%, change: {delta})
- Activation lift: {X}pp (4-week avg: {Y}pp, change: {delta})

## Experiment Results
| Experiment | Hypothesis | Variant | Result | Decision |
|-----------|-----------|---------|--------|----------|
| ...       | ...       | ...     | +3.2% activation (p=0.04) | Adopted |

## Content Health
- Tutorials needing re-recording: {N} (highest priority: {title})
- Coverage gaps identified: {N}
- Videos retired this month: {N}

## Cumulative Autonomous Lift
- Total improvement from AI-run experiments: {X}pp activation lift since Durable start
- Experiments to convergence: {N remaining estimated}

## Recommended Human Actions
1. Re-record '{tutorial title}' — UI has changed since v{X}
2. Record new tutorial for '{topic}' — {N} users searched for this with no match
```

Post to Slack and store in Attio.

### 3. Configure guardrails

Apply the `autonomous-optimization` guardrails specific to this play:

- **Rate limit:** Maximum 1 active experiment at a time across the tutorial library
- **Revert threshold:** If play rate or activation rate drops >20% during an experiment, auto-revert immediately
- **Human approval required for:**
  - Removing a tutorial from the library
  - Changing which tutorials are shown to more than 1 persona
  - Any experiment affecting >50% of new user traffic
- **Cooldown:** After a failed experiment, wait 7 days before testing the same variable
- **Maximum experiments per month:** 4. If all 4 fail, pause and flag for strategic review
- **Content recording:** The agent never records videos. It identifies what needs recording, writes the outline and script, and queues the experiment. A human records and uploads.

### 4. Monitor convergence

The optimization loop runs indefinitely. Track cumulative improvement from AI-run experiments. When 3 consecutive experiments produce <2% improvement on their target metric:

1. The play has reached its local maximum for the current tutorial library, persona mix, and product state
2. Reduce monitoring from daily to weekly
3. Generate a convergence report: current performance levels, all experiments run, cumulative lift, and what would need to change for further improvement (new features, new personas, product redesign)

At convergence, the system enters maintenance mode: weekly health checks, monthly content freshness reviews, and re-activation of daily monitoring only if metrics drop below threshold.

### 5. Evaluate sustainability

After 6 months, assess:

**Primary threshold:** Activation lift has been sustained at >=6pp for the full period (no 2-week window where lift dropped below 4pp)

**Secondary threshold:** Human time investment averaged <=2 hours/week (excluding new video recording)

If PASS: the play is durable. The tutorial library is self-optimizing. Continue in maintenance mode.
If FAIL on sustained lift: the optimization loop is not catching degradation fast enough. Tighten anomaly detection thresholds and reduce the cooldown period.
If FAIL on human time: the agent is escalating too many hypotheses for human review. Raise the risk threshold or expand the set of changes the agent can make autonomously.

## Time Estimate

- Ongoing: ~2 hours/week reviewing briefs and recording new content when flagged
