# Gamified Product Experience — Durable Intelligence

> **Stage:** Product → Retain | **Motion:** Lead Capture Surface | **Channels:** Product

## Outcomes

Gamification participation remains at or above 45% over 6 months without manual intervention. The AI optimization agent continuously finds improvements: adjusting point values, badge thresholds, streak mechanics, leaderboard parameters, and notification timing. The system converges toward the local maximum of gamification engagement for the current user base.

## Leading Indicators

- AI detects anomalies within 24 hours of metric deviation (daily health check fires and classifies correctly)
- Hypotheses generated by the AI produce positive experiment results at least 40% of the time
- Weekly optimization briefs show cumulative positive AI lift trend
- Gamification health monitor triggers fewer escalations over time (system stabilizing)
- Convergence signal: successive experiments produce <2% improvement for 3 consecutive experiments

## Instructions

### 1. Deploy the autonomous optimization loop

Run the `autonomous-optimization` drill configured for the gamification play:

**Phase 1 — Monitor (daily via n8n cron):**
Configure the monitoring loop to track gamification-specific KPIs:
- Primary KPIs: participation rate (rolling 4-week), streak 30-day survival rate, badge earn velocity, leaderboard engagement rate
- Use `posthog-anomaly-detection` to compare last 2 weeks against 4-week rolling average
- Classification: normal (within ±10%), plateau (±2% for 3+ weeks), drop (>20% decline), spike (>50% increase)
- If normal: log to Attio, no action
- If anomaly detected: trigger Phase 2

**Phase 2 — Diagnose (triggered by anomaly):**
Gather gamification context from PostHog and Attio:
- Current segment distribution and per-segment metrics
- Leaderboard health metrics (participation, score gap, top-10 turnover)
- Streak survival curves by streak length
- Badge completion rates by tier
- Recent experiment history
Run `hypothesis-generation` with the anomaly data. Receive 3 ranked hypotheses. Examples:
- "Streak grace period of 1 day is too strict for the Explorer segment — increasing to 2 days will improve 14-day survival by 8pp"
- "Badge velocity is declining because top users have earned all available badges — adding 3 new Gold-tier badges targeting advanced features will restore velocity"
- "Leaderboard engagement dropped because the same 5 users dominate — switching to cohort-based leaderboards will increase new-entrant rate by 30%"
If top hypothesis risk = "high", escalate to human review and stop.

**Phase 3 — Experiment (triggered by hypothesis acceptance):**
Design and launch the experiment:
- Use `posthog-experiments` to create a feature flag splitting traffic between control and variant
- Implement the variant (e.g., adjust grace period, add badges, switch leaderboard scope)
- Minimum duration: 7 days or 100+ users per variant
- Log experiment in Attio: hypothesis, start date, expected duration, success criteria
- Maximum 1 active experiment at a time per the `autonomous-optimization` guardrails

**Phase 4 — Evaluate (triggered by experiment completion):**
Pull results from PostHog. Run `experiment-evaluation`:
- **Adopt**: improvement is statistically significant (95% confidence) and practically meaningful (>3% relative improvement). Roll out to 100%. Log the change.
- **Iterate**: result is directionally positive but not significant. Generate a refined hypothesis. Return to Phase 2.
- **Revert**: variant performed worse or no difference. Restore control. Log the failure. Return to Phase 1.
- **Extend**: insufficient sample size. Continue for another 7 days.
Store evaluation in Attio with decision, confidence level, and reasoning.

**Phase 5 — Report (weekly via n8n cron):**
Generate weekly optimization brief:
- Anomalies detected this week
- Hypotheses generated and their status
- Active experiment progress
- Completed experiments: adopted, reverted, or iterated
- Net metric change from all adopted changes
- Current distance from estimated local maximum
- Recommended focus for next week
Post to Slack and store in Attio.

### 2. Deploy the gamification health monitor

Run the `autonomous-optimization` drill for play-specific monitoring that complements the generic optimization loop:

- Configure daily health checks for all 8 gamification health metrics: participation rate, streak survival, badge velocity, leaderboard view rate, level-up rate, gamified user retention, streak break rate, challenge completion rate
- Implement diagnostic triggers for each declining metric:
  - Participation decline: check onboarding funnel vs existing user dropout
  - Streak survival decline: check day-of-week patterns and grace period usage
  - Badge velocity decline: check if easy badges are exhausted
  - Challenge completion out of range: recalibrate segment difficulty
- Configure automated interventions:
  - Intercom nudge for users who have not discovered gamification after 3+ days
  - Loops streak recovery email for broken streaks >=7 days
  - Intercom leaderboard re-engagement for lapsed leaderboard viewers
- Build weekly gamification health report: aggregate daily checks, trend analysis, intervention outcomes, notable user achievements
- Set escalation rules: any metric critical for 3+ days, retention below 60% for 2 weeks, participation below 30% for 1 week

### 3. Monitor for convergence

The optimization loop runs indefinitely. Watch for convergence signals:

- 3 consecutive experiments produce <2% improvement
- All health metrics stable in "healthy" range for 4+ consecutive weeks
- Weekly optimization briefs consistently report "no anomalies detected"

At convergence:
1. The play has reached its local maximum
2. Reduce monitoring frequency from daily to weekly
3. Reduce experiment cadence from continuous to monthly spot-checks
4. Report: "Gamification engagement is optimized at [current metrics]. Further gains require strategic changes (new game mechanics, new reward types, product-level changes) rather than tactical optimization."

### 4. Handle structural changes

If the AI optimization loop stalls (4 consecutive failed experiments) or health metrics enter critical despite optimization:

**Human action required:** The agent flags this for strategic review. Possible structural changes:
- Add new gamification mechanics (e.g., team challenges, seasonal events, social sharing)
- Redesign the progression system (new levels, prestige mechanics)
- Integrate gamification with external rewards (discounts, swag, premium features)
- Reconsider whether the gamified actions are the right ones to reward

After structural changes are implemented, reset the optimization baseline and restart the loop.
