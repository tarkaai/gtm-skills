# Timing Objection Handling — Baseline Run

> **Stage:** Sales → Connected | **Motion:** Outbound Founder-Led | **Channels:** Direct, Email

## Overview

## Instructions

1. Expand timing objection handling to 20-30 instances over 2 weeks.

2. Create timing objection framework: diagnostic questions, cost of inaction calculator templates, bridging option menu, urgency creation tactics.

3. Build industry-specific cost calculators: quantify typical monthly cost of not solving problem by industry, company size, and pain severity.

4. Set up PostHog event tracking: timing_objection_type, true_vs_false_objection, cost_calculated, urgency_driver_identified, timeline_changed.

5. Develop bridging solutions portfolio: pilot programs, phased rollouts, interim workarounds, quick wins that create momentum toward full purchase.

6. Track timing objection outcomes: measure reengagement rate, time to reactivation, eventual conversion rate, deal size impact of delay.

7. Create urgency playbook: tactics for creating legitimate urgency (end-of-quarter discounts, implementation timelines, seasonal preparation, regulatory deadlines).

8. Set pass threshold: Timing objections addressed on ≥80% of instances over 2 weeks with ≥50% resulting in timeline acceleration or bridging solution acceptance.

9. Analyze timing patterns: which timing objections are real constraints vs. smokescreens, which urgency drivers work best, which bridging options convert highest.

10. If threshold met, document timing objection playbook and proceed to Scalable; if not, refine diagnostic questions or urgency tactics.
